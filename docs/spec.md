## 3. 业务场景与边界条件

### 场景 1：进入系统 — 地图上查看推荐车辆分布

**正常路径：**
用户登录后，传入当前经纬度和可视范围，系统返回该区域内推荐车辆的分布信息（含坐标、评分、数量，有数量上限）。

**接口：** `GET /api/vehicles/map?lat={lat}&lng={lng}&radius={radius}&limit={limit}`

**边界条件：**

| 异常场景 | 处理方式 |
|:---|:---|
| 用户未登录或 Token 过期 | 返回 401，提示"请先登录" |
| 传入经纬度为空或格式非法 | 返回 400，提示"经纬度参数不合法" |
| 可视范围内无可用车辆 | 返回 200，data 为空数组，提示"当前区域暂无可用车辆"，建议扩大搜索范围 |
| 可视范围超出系统服务区域 | 返回 400，提示"超出服务范围" |
| radius 超过最大限制（如 >20km） | 自动截断为最大值，返回截断后的结果 |
| limit 未传或超过上限 | 默认值 20，最大值 50，超出自动截断 |
| Redis 缓存中的车辆位置数据过期 | 回源 MySQL 查询，同时异步刷新缓存 |

---

### 场景 2：查看推荐车辆列表

**正常路径：**
用户不看地图，仅查看文字列表形式的推荐车辆信息（车辆编号、电量、距离、评分等，不含坐标）。

**接口：** `GET /api/vehicles/recommend?lat={lat}&lng={lng}&limit={limit}`

**边界条件：**

| 异常场景 | 处理方式 |
|:---|:---|
| 用户未登录或 Token 过期 | 返回 401，提示"请先登录" |
| 传入经纬度为空或格式非法 | 返回 400，提示"经纬度参数不合法" |
| 当前区域无可用（AVAILABLE）车辆 | 返回 200，data 为空数组，提示"附近暂无可用车辆" |
| 所有附近车辆电量均低于阈值（如 <10%） | 返回 200，data 为空数组，提示"附近车辆电量不足，请稍后再试" |
| 推荐引擎计算异常（策略执行失败） | 降级为默认策略（仅按距离排序），记录告警日志 |
| limit 未传 | 默认值 10 |

---

### 场景 3：选中车辆 → 创建租赁订单

**正常路径：**
用户在列表或地图上选中某辆车，点击租赁，系统校验车辆状态后预占车辆并创建订单。

**接口：** `POST /api/orders/create` body: `{vehicleId, startLat, startLng}`

**边界条件：**

| 异常场景 | 处理方式 |
|:---|:---|
| 用户未登录或 Token 过期 | 返回 401，提示"请先登录" |
| 车辆 ID 不存在 | 返回 404，提示"车辆不存在" |
| 车辆状态非 AVAILABLE（已被他人预占/租赁中/充电/维修） | 返回 409，提示"该车辆当前不可租用"，附带车辆当前状态 |
| 车辆电量低于阈值（如 <10%） | 返回 400，提示"车辆电量不足，请选择其他车辆" |
| 车辆健康度低于阈值（如 <60） | 返回 400，提示"车辆状态异常，请选择其他车辆" |
| 用户已有未完成的租赁订单 | 返回 409，提示"您有一笔进行中的订单，请先完成"，返回已有订单号 |
| Redis 预占锁获取失败（并发冲突） | 返回 409，提示"该车辆正在被其他用户选择中"，触发自旋重试推荐相似车辆 |
| MySQL 乐观锁校验失败（并发写入冲突） | 自动重试 1 次，仍失败则返回 409，提示"操作冲突，请重试" |
| Redis 服务不可用 | 降级为仅 MySQL 乐观锁校验，记录告警日志 |
| 订单号生成重复（极端并发） | 重试生成，最多 3 次，仍失败返回 500 |
| 传入经纬度为空或超出服务区域 | 返回 400，提示"起点位置参数不合法" |

---

### 场景 4：确认订单 → 进入租赁状态

**正常路径：**
订单创建后处于 CREATED 状态（预占中），用户确认后订单变为 RUNNING，车辆状态从 RESERVED 变为 RENTING。

**接口：** `POST /api/orders/{orderId}/confirm`

**边界条件：**

| 异常场景 | 处理方式 |
|:---|:---|
| 订单不存在 | 返回 404，提示"订单不存在" |
| 订单不属于当前用户 | 返回 403，提示"无权操作此订单" |
| 订单状态非 CREATED（已确认/已取消/已完成） | 返回 409，提示"订单状态不允许确认"，附带当前状态 |
| 预占锁已过期（用户超时未确认） | 订单自动标记为 CANCELLED，车辆回退为 AVAILABLE，返回 410 提示"预占已过期，请重新选择车辆" |
| 车辆在预占期间被管理员标记为维修/充电 | 订单自动取消，返回 409 提示"车辆状态已变更，无法确认" |
| 确认请求重复提交 | 幂等处理，已确认的订单直接返回当前订单信息 |
| 确认过程中车辆状态被并发修改（乐观锁冲突） | 返回 409，提示"操作冲突，请重试" |

---

### 场景 5：一键租赁（自动推荐 + 自动预占 + 创建订单 → 用户确认）

**正常路径：**
用户传入当前位置，系统自动执行：评分推荐最优车辆 → Redis 预占 → 创建订单 → 返回订单信息供用户确认。

**接口：** `POST /api/orders/quick-rent` body: `{lat, lng}`

**边界条件：**

| 异常场景 | 处理方式 |
|:---|:---|
| 用户未登录或 Token 过期 | 返回 401，提示"请先登录" |
| 当前无可用车辆 | 返回 200，code=NO_VEHICLE，提示"附近暂无可用车辆，请稍后再试" |
| 推荐的第一辆车预占失败（被并发抢走） | 自动尝试第二辆，最多尝试 N 辆（如 3 辆），全部失败则返回提示"车辆繁忙，请稍后重试" |
| 用户已有未完成的租赁订单 | 返回 409，提示"您有一笔进行中的订单，请先完成" |
| 推荐引擎异常 | 返回 500，提示"系统繁忙，请稍后重试" |
| 预占成功但订单创建失败 | 释放预占锁，车辆回退为 AVAILABLE，返回 500 提示"订单创建失败，请重试" |
| 一键租赁返回的订单，用户长时间未确认 | 预占锁过期后自动取消，用户再次查询时提示"订单已过期" |
| 传入经纬度为空或超出服务区域 | 返回 400，提示"位置参数不合法" |

---

### 场景 6：通用边界条件（贯穿所有场景）

| 异常场景 | 处理方式 |
|:---|:---|
| 请求频率过高（同一用户短时间内大量请求） | 返回 429，提示"请求过于频繁，请稍后再试"（限流策略） |
| 服务内部异常（数据库/Redis/MQ 不可用） | 返回 500，统一错误格式，记录完整错误日志（含 traceId） |
| 并发操作导致数据不一致 | 乐观锁 + 重试机制兜底，最终一致性保障 |
| 用户 Token 被篡改 | 返回 401，提示"认证失败" |
| 请求参数缺失必填字段 | 返回 400，提示具体缺失字段 |
