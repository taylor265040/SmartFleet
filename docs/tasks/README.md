# SmartFleet 任务分解文档

本文档将SmartFleet智能车辆租赁管理平台的开发任务分解为7个主要模块（M1-M7），每个模块按照统一的开发流程进行分解：**工程准备 → 测试先行 → 硬编码跑通 → 骨架 → 数据加载 → 检索实现 → 集成具体实现**。

## 任务模块概览

| 模块 | 名称 | 主要内容 | 依赖项 |
|------|------|----------|--------|
| M1 | 基础设施搭建 | Spring Boot + MyBatis-Plus项目骨架、MySQL/Redis/RocketMQ配置、统一日志和异常处理 | 无 |
| M2 | 用户认证与安全 | Spring Security + JWT、用户注册/登录/Token刷新、基于角色的访问控制 | M1 |
| M3 | 智能车辆调度引擎 | 多维加权评分引擎、策略模式、地图车辆推荐接口 | M1, M2 |
| M4 | 高并发租赁一致性控制 | Redis预占锁、MySQL乐观锁、自旋重试、订单全流程接口 | M1, M2, M3 |
| M5 | 车辆状态机管理 | 状态机、状态转移矩阵校验、Redis缓存同步 | M1, M4 |
| M6 | 区域运力调度与动态定价 | GeoHash网格化、供需比计算、动态定价引擎 | M1, M3, M4 |
| M7 | 实时数据处理与监控 | Cache Aside、RocketMQ异步处理、WebSocket推送、监控API | M1, M2, M3, M4, M5, M6 |

## 开发流程说明

每个模块的开发都遵循以下流程：

### 1. 工程准备
- 初始化项目结构
- 添加依赖
- 配置基础参数

### 2. 测试先行
- 编写单元测试
- 编写集成测试
- 定义验收标准

### 3. 硬编码跑通
- 实现核心逻辑（硬编码）
- 验证基本功能
- 快速原型验证

### 4. 骨架
- 创建接口和实现类
- 设计类结构
- 定义API规范

### 5. 数据加载
- 实现数据持久化
- 配置缓存策略
- 数据初始化

### 6. 检索实现
- 实现查询接口
- 优化查询性能
- 实现分页和排序

### 7. 集成具体实现
- 集成各模块
- 端到端测试
- 性能优化

## 测试用例说明

每个模块的`test-record`部分包含三类测试用例：

### 1. 正常测试用例
- 验证预期功能正常工作
- 覆盖主要业务流程
- 验证输入输出正确性

### 2. 边界测试用例
- 测试边界条件
- 验证参数校验
- 测试异常输入处理

### 3. 失败测试用例
- 测试错误处理
- 验证异常情况
- 测试系统容错能力

## 验收标准

### 接口验收
- 所有API接口正常工作
- 返回正确的状态码和响应格式
- 参数校验和错误处理正确

### 单元测试验收
- 各Service核心方法覆盖率 ≥ 80%
- 状态机所有合法/非法跃迁路径均有测试用例
- 评分引擎各策略权重计算有测试验证
- 一键租赁的自动重试逻辑有测试覆盖

### 集成测试验收
- 核心租赁流程端到端测试通过
- 一键租赁流程端到端测试通过
- 高并发场景无超卖
- 预占过期自动回退测试通过

## 开发顺序建议

### 第一阶段：基础设施（M1）
1. 搭建Spring Boot项目骨架
2. 配置MySQL、Redis、RocketMQ
3. 建立统一日志和异常处理框架

### 第二阶段：核心功能（M2, M3）
1. 实现用户认证与安全（M2）
2. 实现智能车辆调度引擎（M3）

### 第三阶段：业务逻辑（M4, M5）
1. 实现高并发租赁一致性控制（M4）
2. 实现车辆状态机管理（M5）

### 第四阶段：高级功能（M6, M7）
1. 实现区域运力调度与动态定价（M6）
2. 实现实时数据处理与监控（M7）

## 注意事项

1. **依赖管理**：确保各模块依赖关系正确，避免循环依赖
2. **测试覆盖**：每个模块都要有完整的测试用例，包括正常、边界、失败场景
3. **代码质量**：遵循编码规范，保持代码可读性和可维护性
4. **文档更新**：及时更新API文档和开发文档
5. **版本控制**：使用Git进行版本管理，每个功能模块一个分支
